# Go Wire DI Setup

Generates `cmd/server/wire.go` with provider sets and an injector function so Google Wire can auto-generate `wire_gen.go`.

**Input**: List of components and their constructors (or read from main.go), module path
**Output**: `cmd/server/wire.go`, `cmd/server/wire_gen.go` scaffold, Makefile target

## Trigger phrases
- "set up Wire"
- "dependency injection in Go"
- "wire providers"
- "google wire setup"
- "generate wire.go"
- "DI for my Go service"
- "my main.go is getting too large to manage dependencies"

## Output files
- `cmd/server/wire.go` (provider sets + injector)
- `cmd/server/app.go` (App struct + NewApp)
- Updated `main.go` snippet
- Updated `Makefile` with `wire-gen` target

## Workflow

### STEP 1 — Identify components
If the user provides their `main.go`, read it and extract each `New<X>(deps...)` constructor and the dependency graph. Otherwise ask the user to list components.

### STEP 2 — Generate `cmd/server/wire.go`

```go
//go:build wireinject
// +build wireinject

package main

import "github.com/google/wire"

var RepositorySet = wire.NewSet(
    pkgpostgres.MustConnect,
    postgres.NewListingRepo,
    wire.Bind(new(repository.ListingRepository), new(*postgres.ListingRepo)),
)

var ServiceSet = wire.NewSet(
    service.NewListingService,
    wire.Bind(new(service.ListingService), new(*service.listingServiceImpl)),
)

var HandlerSet = wire.NewSet(handler.NewListingHandler)

func InitializeApp(cfg *config.Config) (*App, error) {
    wire.Build(RepositorySet, ServiceSet, HandlerSet, NewApp)
    return nil, nil
}
```

### STEP 3 — Generate `cmd/server/app.go`

```go
type App struct {
    GRPCServer *grpc.Server
    HTTPServer *http.Server
    DB         *sqlx.DB
}

func NewApp(grpcServer *grpc.Server, httpServer *http.Server, db *sqlx.DB) *App {
    return &App{GRPCServer: grpcServer, HTTPServer: httpServer, DB: db}
}
```

### STEP 4 — Show `wire_gen.go` scaffold (what Wire generates)

```go
// Code generated by Wire. DO NOT EDIT.
//go:build !wireinject

func InitializeApp(cfg *config.Config) (*App, error) {
    db := postgres.MustConnect(cfg.DatabaseURL)
    listingRepo := postgres.NewListingRepo(db)
    listingService := service.NewListingService(listingRepo)
    listingHandler := handler.NewListingHandler(listingService)
    return NewApp(...), nil
}
```

### STEP 5 — Update Makefile

```makefile
wire-gen:
    wire ./cmd/server/...
    go build ./cmd/server/...

wire-check:
    wire check ./cmd/server/...
```

### STEP 6 — Update `main.go`

```go
func main() {
    cfg := config.MustLoad()
    app, err := InitializeApp(cfg)
    if err != nil { log.Fatal("init app:", err) }
    app.Run()
}
```

### STEP 7 — Write all files and `present_files`

## Quality checklist
- [ ] `//go:build wireinject` tag present — prevents double-compilation
- [ ] Every interface has a `wire.Bind` call
- [ ] Provider sets group by layer (repo/service/handler)
- [ ] `InitializeApp` returns `error` — providers that can fail bubble up

## Edge cases
- **Circular dependency**: split the component causing the cycle
- **Provider returns error**: wire handles `func() (T, error)` natively
- **User prefers manual DI**: generate a manual `initApp()` function in main.go instead
- **Multiple entry points**: generate separate injectors per `cmd/` directory
