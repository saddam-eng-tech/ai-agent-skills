# Issue Finder Skill

Discovers real developer problems from the web and filters them into a
prioritised list of issues that are good candidates for solving with a
Claude SKILL.md automation.

---

## What counts as a "skill-solvable issue"?

A developer issue is a good skill candidate if it meets **3 or more** of
these criteria:

| Criterion | Why it matters |
|-----------|---------------|
| Repetitive / tedious | Automation ROI is high |
| Involves file I/O (docx, pdf, xlsx, pptx, code) | Claude's file tools handle this natively |
| Requires reasoning + structure | Pure scripts can't do it, but Claude can |
| Comes up frequently in forums (many upvotes/replies) | Validated demand |
| Solvable without external auth / databases | Keeps skill self-contained |
| Time-consuming but formulaic | High value to automate |

Issues that are **not** good candidates:
- Require real-time data (stock prices, live APIs)
- Depend on user-specific infrastructure (CI/CD pipelines, private repos)
- Are already solved by mature CLI tools
- Need a GUI / browser automation at the core

## Step-by-Step Workflow

### STEP 1 — Gather search targets

Run web searches across **at least 4** sources using `web_search` and `web_fetch`.

**Search queries to run:**
```
site:reddit.com/r/webdev "I hate having to manually"
site:reddit.com/r/programming "wish there was a tool"
site:news.ycombinator.com "tedious" developer workflow
site:stackoverflow.com "is there a way to automate" 2024 OR 2025
site:github.com/issues "feature request" "generate" OR "automate"
"developer workflow" "painful" OR "annoying" site:dev.to
```

Run **6–8 searches**. Fetch 2–3 full pages for depth.

---

### STEP 2 — Extract raw issues

For each complaint/request found, capture:
```
RAW ISSUE:
- Source URL
- Title / quote
- Upvotes or engagement (if visible)
- Core pain: (one sentence)
```
Collect at least **15–25 raw issues** before filtering.

---

### STEP 3 — Score and filter

Score each issue against the criteria table. Award 1 point per criterion met.
Keep only issues scoring **3+**.

For each kept issue, write:
```markdown
### Issue: [Short title]
- **Score**: X/6
- **Source**: [URL]
- **Pain summary**: ...
- **Frequency signal**: ...
- **Skill opportunity**: ...
- **Input**: ...
- **Output**: ...
- **Complexity estimate**: Low / Medium / High
```

---

### STEP 4 — Rank and present

Sort by: Score → Frequency → Complexity (Low first).

Present with a summary table at the top.

---

### STEP 5 — Output format

Save to `issue-list-YYYY-MM-DD.md`. End with:
```
*Generated by issue-finder skill. Feed this list into the
`issue-skill-generator` skill to auto-generate SKILL.md files.*
```
